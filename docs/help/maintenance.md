# Maintenance — Requests & Work Orders

## Overview

This guide explains how tenants submit maintenance requests and how staff manage, assign, and close work orders.

## Quick Links

- Tenant submit: `/tenant/maintenance`
- Staff board: `/dashboard/maintenance`

## Submitting a Request (Tenant)

Where: `/tenant/maintenance` → New Request

Steps:

1. Choose the property and unit.
2. Provide a clear title and description of the issue.
3. Set urgency/priority and attach photos if available.
4. Submit and wait for staff assignment and updates.

Image placeholder: `/public/help-assets/tenant-maintenance-01.png`

## Managing Requests (Staff)

Where: `/dashboard/maintenance`

Steps:

1. Review new requests in the queue or Kanban board.
2. Assign to a technician or external contractor.
3. Add notes, schedule visits, and attach images or invoices.
4. Update status through lifecycle: Requested → Assigned → In Progress → Completed.

Image placeholder: `/public/help-assets/admin-maintenance-01.png`

## Scheduling and SLA

- Use priority to triage and assign realistic SLAs.
- For recurring maintenance, create recurring tasks or calendar entries.

## Closing & Documentation

- Add final notes and attach photos before marking a request as completed.
- Close requests only after tenant confirmation when required.

## Troubleshooting

- Missing attachments: ask the tenant to re-upload or email images to support.
- Request not visible: check filters and permission scopes.
